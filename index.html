SpriteAlive

Java, Spring

SpriteAlive Site

In beading video game sprites, I found it quite difficult to keep track of the colors I used throughout a sprite, and their respective places. During Spring break of 2021 I looked around but was unsatisfied with other "picture to bead sprite" converters and managers online. So, I wanted to use my new Java skills I learned from class. I was using Google Sheets to keep track of my bead colors, so I decided to write a Java program that would work with Google Sheets Apps Script that did all the tedious work for me.


In fuse beading video game sprites, I found it pretty hard to keep track of all the colors and positions- especially when beading images with as much variation in color as sprites. The goal for this project was to fill this need, so I can make more perler designs faster. I applied programming concepts I learned from class for the Java backend. I used Google Sheets to organize my beads already, so I made an extension with Apps Script.


This is really just a project for learning, since the codebase is pretty amateurish and only uses the Spring framework's embedded Tomcat server for deployment onto the Google Cloud platform. After I become more acclimated to standard web app practices at my university, I hope to do a full rewrite of this program later- as well as release it to the public, for others to use!

Blueguy vs The World - Game Builder Garage

Game Builder Garage, Nintendo Switch

Game ID: G-005-FPP-T4V

This was a short game I created after the release of Nintendo's Game Builder Garage. It was fun learning a new "toy" engine- GBG is pretty capable when you want to create fun, simple projects or prototypes. I love the integration with the Switch's hardware, essentially creating this 'sandbox' where you can mess around with the various sensors the JoyCon gives you.


The thing I disliked about it was, truthfully, how limiting it was. This game takes up the entire Nodon (the fun "code blocks" you use to program your games) limit, which is 512. Imagine, you have to keep track of game state, level progression, user input, environment, and of course, game logic, under this limit. I found myself trying too hard to just make things work; sometimes I'd rather just use Unity.


In the end, GBG is still a game, about making games. In Japan, it seems to be advertised for kids who want to get their feet wet in coding- NOT for people wanting to make a 3d metroidvania. Despite this, I had fun nonetheless. The community is still very much alive, with a creator named VideoDojo and the GBG Discord. People there smarter than me really push the engine to its limits.

Perler Bead Projects
